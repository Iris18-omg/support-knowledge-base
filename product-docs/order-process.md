# Спецификация процесса: оформление заказа в мобильном приложении

В документе приведены верхнеуровневое описание основного бизнес-процесса фудтех-платформы здорового питания и пользовательские сценарии оформления заказа через мобильное приложение.

---

## 1. Анализ деятельности платформы

### 1.1 Краткое описание системы
Омниканальная фудтех-платформа, специализирующаяся на розничной продаже и экспресс-доставке продуктов здорового питания. 

Взаимодействие с клиентами осуществляется через три основных канала:
* Физическая сеть магазинов (офлайн).
* Официальный веб-сайт (онлайн-витрина, личный кабинет, корпоративные проекты, контакты).
* Мобильное приложение (основной канал e-commerce: каталог, интерактивная карта, программа лояльности, поддержка в чате).

### 1.2 Верхнеуровневая модель бизнес-процесса (SIPOC)

| Supplier (Поставщик) | Input (Вход) | Process (Процесс) | Output (Выход) | Customer (Заказчик) |
| :--- | :--- | :--- | :--- | :--- |
| Поставщики продуктов питания,<br>IT-департамент (данные каталога) | Потребность клиента в продуктах<br>Товарные остатки на складах/в дарксторах | **Обеспечение покупателя продуктами питания через онлайн и офлайн-каналы:**<br>выбор товаров, формирование корзины, сборка, оплата и доставка заказа. | Получение клиентом собранного заказа. | Конечный розничный покупатель. |

## 2. Сценарии процесса заказа в мобильном приложении

### 2.1 Основной сценарий 
1. Пользователь переходит в раздел **«Каталог»**.
2. Система отображает дерево категорий товаров и строку поиска.
3. Пользователь выбирает нужную категорию и подкатегорию.
4. Система выводит список релевантных товаров.
5. Пользователь выбирает конкретный товар.
6. Система открывает карточку товара с подробным описанием и характеристиками.
7. Пользователь нажимает кнопку **«В корзину»**.
8. Система обновляет счётчик товаров и общую сумму в корзине.
9. Пользователь переходит на экран **«Корзина»**.
10. Система отображает перечень выбранных позиций, условия и доступные интервалы доставки.
11. Пользователь нажимает кнопку **«Перейти к оплате»**.
12. Система открывает шлюз выбора платежного метода.

### 2.2 Альтернативные сценарии 

#### 2.2.1 Сценарий А: Искомый товар отсутствует
1. Пользователь вводит наименование товара в строку поиска.
2. Система осуществляет поиск по базе данных, совпадений не обнаружено.
3. Система отображает заглушку с текстом: *«Товар не найден»*.
4. Пользователь возвращается к экрану поиска или навигации по категориям.

#### 2.2.2 Сценарий Б: Удаление одной позиции из корзины
1. Пользователь находится на экране **«Корзина»**.
2. Пользователь нажимает иконку удаления (или уменьшает количество товара до 0).
3. Система временно скрывает товар и запускает таймер отмены действия (снакбар с кнопкой *«Вернуть»*).
4. Пользователь не совершает действий до истечения таймера.
5. Система окончательно удаляет позицию, пересчитывает общую сумму заказа и обновляет интерфейс.

#### 2.2.3 Сценарий В: Полная очистка корзины
1. Пользователь находится в экране **«Корзина»**.
2. Пользователь нажимает кнопку **«Очистить всё»**.
3. Система удаляет все товары из текущей сессии корзины.
4. Система возвращает пользователя в каталог.
