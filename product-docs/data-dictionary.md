# Словарь сущностей модуля оформления заказа

В данном документе приведены и описаны ключевые системные сущности (классы объектов), задействованные в процессах выбора, хранения и оформления заказа продуктов питания.

| Сущность (Класс) | Системное описание | Бизнес-смысл сущности |
| :--- | :--- | :--- |
| **User** (Пользователь) | Авторизованный аккаунт клиента в системе. | Покупатель, инициирующий и оформляющий заказ. |
| **Catalog** (Каталог) | Справочник ассортимента магазина. | Иерархический перечень всех товаров, доступных к покупке. |
| **Category** (Категория) | Объект верхнего уровня иерархии справочника товаров. | Крупное логическое объединение товаров (напр., *«Молочные продукты»*). |
| **Subcategory** (Подкатегория) | Дочерний объект ветки Category (Категория). | Узкая группа товаров внутри категории (напр., *«Йогурты»*). |
| **Product** (Товар) | Конечная товарная единица в базе данных. | Продукт питания, обладающий ценой, весом, составом и остатком. |
| **Cart** (Корзина) | Временный объект хранения выбранных товаров. | Набор товаров, которые пользователь планирует приобрести в рамках текущей сессии. |
| **Order** (Заказ) | Итоговый зафиксированный объект сделки с уникальным ID. | Оформленная покупка пользователя, переданная на сборку и доставку. |
| **OrderComment** (Комментарий) | Текстовый атрибут, привязанный к объекту Order (Заказ). | Дополнительные текстовые инструкции для сборщика или курьера. |
| **DeliveryAddress** (Адрес) | Гео-объект, связанный с профилем User (Пользователь). | Географическая точка и параметры (этаж, код двери) получения заказа. |
| **PromoCode** (Промокод) | Системный объект правил лояльности и скидок. | Инструмент пересчёта финальной стоимости товаров при выполнении условий. |
| **PaymentMethod** (Способ оплаты) | Интеграционный платежный токен. | Выбранный пользователем инструмент проведения финансовой транзакции. |
